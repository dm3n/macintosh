# Local Development System (Mac)

Macintosh starts on the Mac. This layer is your command center for code, context, and operator control.

## Purpose

- provide a fast, repeatable development environment
- keep all coding agents aligned to one operating model
- keep long-term memory available inside every session

## Terminal Roles

- **Superset**: primary terminal for coding sessions and agent-driven implementation
- **Warp**: general terminal for broader shell workflows and non-coding operations

This split keeps development ergonomics intentional: implementation in Superset, general ops in Warp.

## Local Agent System

Four coding agents share one context model:
- Claude Code
- Codex CLI
- Gemini CLI
- OpenCode

Shared guarantees:
- same global standards and project context
- same superpowers baseline
- same Brain vault references and memory loop

## Knowledge Backbone (PKB)

The Brain vault is the persistent memory substrate for local work:
- memory index and operating notes
- compiled wiki entities/concepts/SOPs
- session history and project references
- raw-to-structured processing pipeline

Result: decisions and insights survive across tools, sessions, and models.

## Local Responsibilities

This layer owns:
- product implementation and debugging
- documentation and system design updates
- quick checks, scripts, and repo operations
- context synthesis from PKB before major decisions

This layer does **not** own long-running infrastructure execution. That belongs to personal cloud/homelab nodes.
